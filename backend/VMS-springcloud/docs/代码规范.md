# 后端代码规范

## 1. 包命名规范

所有模块的包路径必须统一为：`cn.yznu.vms.{module_name}`

- **Group ID**: `cn.yznu.vms` (学号/项目标识)
- **Artifact ID**: `vms-{module_name}`
- **Package**: `cn.yznu.vms.{module_name}`

**示例**：

- 用户模块: `cn.yznu.vms.user`
- 视频模块: `cn.yznu.vms.video`
- 公共模块: `cn.yznu.vms.common`

---

## 2. Controller 规范

### 2.1 响应格式

所有接口必须返回 `Result<T>` 类型：

```java
@GetMapping("/example")
public Result<ExampleVO> example() {
    return Result.success(new ExampleVO());
}
```

### 2.2 异常处理

**禁止**直接返回 `Result.fail()`，必须抛出 `BusinessException`，由全局异常处理器统一处理。

**✅ 正确示范**：

```java
if (user == null) {
    throw new BusinessException(ResultCode.USER_NOT_FOUND);
}
```

**❌ 错误示范**：

```java
if (user == null) {
    return Result.fail("用户不存在");
}
```

### 2.3 接口文档

所有 Controller 类和方法必须添加 Swagger 注解：

- 类：`@Tag`
- 方法：`@Operation`
- 参数：`@Parameter`

---

## 3. Service 规范

### 3.1 事务管理

涉及**多表更新**或**级联删除**的方法，必须添加 `@Transactional` 注解，并指定回滚异常类型。

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void deleteWithRelations(Long id) {
    // 逻辑...
}
```

### 3.2 业务校验

Service 层负责业务逻辑校验，校验失败直接抛出 `BusinessException`。

---

## 4. 枚举规范

### 4.1 状态枚举

所有状态类枚举（如 `StatusEnum`）应序列化为**整数值**（0/1）而非中文描述，以兼容前端逻辑。

```java
@EnumValue
@JsonValue
private final Integer value; // 序列化为 0 或 1
```

---

## 5. 配置文件规范

### 5.1 MyBatis-Plus

`type-aliases-package` 必须指向正确的实体类包路径：

```yaml
mybatis-plus:
  type-aliases-package: cn.yznu.vms.{module}.entity
```

### 5.2 日志级别

Spring Cloud Gateway 或特定模块如需调试，可调整日志级别：

```yaml
logging:
  level:
    cn.yznu.vms: debug
```
