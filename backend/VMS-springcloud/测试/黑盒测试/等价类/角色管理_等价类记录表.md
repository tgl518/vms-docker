# 角色管理模块 - 等价类划分表

**模块**：角色管理 | **日期**：2026-01-07 | **测试方法**：等价类划分法

---

## 一、创建角色 (POST /user/role)

| 编号 | 字段 | 有效等价类 | 无效等价类 |
|------|------|------------|------------|
| EC01 | code | 1-50字符的唯一编码 | 空/null |
| EC02 | code | - | 已存在的编码 |
| EC03 | name | 1-100字符的唯一名称 | 空/null |
| EC04 | name | - | 已存在的名称 |
| EC05 | description | null 或 有效字符串 | - |
| EC06 | sort | null 或 整数 | - |
| EC07 | enable | true/false/null | - |
| EC08 | permissionIds | 空列表或有效ID | - |

---

## 二、更新角色 (PATCH /user/role/{id})

| 编号 | 条件 | 有效等价类 | 无效等价类 |
|------|------|------------|------------|
| EC09 | id | 存在的普通角色ID | 不存在的ID |
| EC10 | name | 不重复的新名称 | 与其他角色重复 |
| EC11 | 系统角色 | 普通角色可禁用 | SUPER_ADMIN/admin不可禁用 |
| EC12 | 权限 | 普通角色可清空 | 系统角色不可清空 |

---

## 三、删除角色 (DELETE /user/role/{id})

| 编号 | 条件 | 有效等价类 | 无效等价类 |
|------|------|------------|------------|
| EC13 | id | 存在的ID | 不存在的ID |
| EC14 | 角色类型 | 普通角色 | 系统角色(SUPER_ADMIN/admin) |
| EC15 | 用户关联 | 无用户关联 | 有用户关联 |

---

## 四、查询角色 (GET /user/role/{id} 和 /page)

| 编号 | 条件 | 有效等价类 | 无效等价类 |
|------|------|------------|------------|
| EC16 | id | 存在的角色ID | 不存在的ID |
| EC17 | pageNo | 正整数 | 0或负数 |
| EC18 | name过滤 | 能匹配的名称 | 无匹配 |

---

**统计**：18个等价类
