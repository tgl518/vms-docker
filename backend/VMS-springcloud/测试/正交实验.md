# 🧪 正交实验设计 (Orthogonal Experiment Design)

> **实验目的**: 通过正交表 $L_9(3^4)$ 科学地减少测试用例数量，同时保证对系统兼容性和环境鲁棒性的最大覆盖。
> **实验对象**: VMS 用户管理模块 (User Management Module)

---

## 1. 因子与水平定义 (Factors & Levels)

我们选取了对 Web 系统影响最大的 4 个因子，每个因子定义 3 个水平：

| 因子 (Factor) | 水平 1 (Level 1) | 水平 2 (Level 2) | 水平 3 (Level 3) |
| :--- | :--- | :--- | :--- |
| **A: 操作系统** | Windows 11 | macOS 14 | Linux (Ubuntu) |
| **B: 浏览器** | Chrome (Blink) | Firefox (Gecko) | Edge (Chromium) |
| **C: 网络环境** | WIFI (稳定) | 4G (高延迟) | 弱网 (丢包率 20%) |
| **D: 缓存状态** | 无缓存 (Cold) | 有缓存 (Warm) | 缓存过期 (Expired) |

---

## 2. 正交测试矩阵 (Orthogonal Matrix)

根据 $L_9(3^4)$ 正交表生成的 9 条核心测试用例：

| 用例 ID | A: 操作系统 | B: 浏览器 | C: 网络环境 | D: 缓存状态 | 预期结果 (Expected) | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **T01** | Windows 11 | Chrome | WIFI | 无缓存 | ✅ Pass | 基准测试环境 |
| **T02** | Windows 11 | Firefox | 4G | 有缓存 | ✅ Pass | 测试移动端热点兼容性 |
| **T03** | Windows 11 | Edge | 弱网 | 缓存过期 | ⚠️ Slow | 验证超时重试机制 |
| **T04** | macOS 14 | Chrome | 4G | 缓存过期 | ✅ Pass | 验证缓存击穿/穿透逻辑 |
| **T05** | macOS 14 | Firefox | 弱网 | 无缓存 | ⚠️ Slow | 验证页面加载 Loading 态 |
| **T06** | macOS 14 | Edge | WIFI | 有缓存 | ✅ Pass | 验证缓存读取速度 |
| **T07** | Linux | Chrome | 弱网 | 有缓存 | ⚠️ Slow | 验证弱网下的静态资源加载 |
| **T08** | Linux | Firefox | WIFI | 缓存过期 | ✅ Pass | 验证 Linux 字体渲染与缓存刷新 |
| **T09** | Linux | Edge | 4G | 无缓存 | ✅ Pass | 验证跨平台兼容性 |

---

## 3. 结果分析方法

执行上述 9 个用例后，应重点关注：
1.  **极差分析**: 哪个因子对系统性能/稳定性的影响最大（例如：是否"弱网"的影响远大于"浏览器差异"）。
2.  **缺陷定位**: 如果 T03, T05, T07 均失败，说明系统对 **"弱网"** 环境的处理能力不足（如缺乏重试机制或超时设置过短）。

---

| **R03** | ✅ 有效 | ❌ 为空 | (任意) | 提示: "请输入密码" |
| **R04** | ✅ 有效 | ✅ 有效 | ❌ 错误 | 提示: "验证码错误" |
