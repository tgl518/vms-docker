# 角色管理模块 - 黑盒测试用例

> 测试对象：角色管理功能 (RBAC)  
> 测试方法：等价类划分、边界值分析、判定表  

---

## 1. 等价类测试用例

### 1.1 创建角色 - 等价类划分

| 输入项 | 有效等价类 | 无效等价类 |
|--------|-----------|-----------|
| **角色编码 (code)** | EC1: 1-50字符的英文字母/下划线 | EC2: 空字符串<br>EC3: 超过50字符<br>EC4: 包含特殊字符<br>EC5: 与已有编码重复 |
| **角色名称 (name)** | EC6: 1-50字符的中英文 | EC7: 空字符串<br>EC8: 超过50字符<br>EC9: 与已有名称重复 |
| **排序 (sort)** | EC10: 0-999的正整数 | EC11: 负数<br>EC12: 超过999 |
| **启用状态 (enable)** | EC13: true<br>EC14: false | EC15: null (使用默认值) |
| **权限列表 (permissionIds)** | EC16: 有效权限ID列表 | EC17: 空列表<br>EC18: 包含不存在的权限ID |

### 1.2 创建角色 - 等价类测试用例表

| 用例编号 | 测试目标 | code | name | sort | enable | permissionIds | 预期结果 |
|---------|---------|------|------|------|--------|---------------|---------|
| TC-EQ-01 | 有效等价类组合 | "EDITOR" | "编辑员" | 10 | true | [1,2,3] | **成功创建** |
| TC-EQ-02 | 编码为空 | "" | "测试" | 1 | true | [1] | **失败**: 编码不能为空 |
| TC-EQ-03 | 编码重复 | "SUPER_ADMIN" | "新角色" | 1 | true | [1] | **失败**: 编码已存在 |
| TC-EQ-04 | 名称为空 | "TEST" | "" | 1 | true | [1] | **失败**: 名称不能为空 |
| TC-EQ-05 | 名称重复 | "NEW_CODE" | "超级管理员" | 1 | true | [1] | **失败**: 名称已存在 |
| TC-EQ-06 | 排序为负数 | "TEST" | "测试" | -1 | true | [1] | **失败**: 排序必须>=0 |
| TC-EQ-07 | 启用状态为false | "GUEST" | "访客" | 99 | false | [1] | **成功创建** (状态禁用) |
| TC-EQ-08 | 权限列表为空 | "VIEWER" | "浏览者" | 5 | true | [] | **成功创建** (无权限) |

## 5. 场景法测试 - 创建角色全流程

### 5.1 场景描述

```
基本流程：
1. 管理员进入角色管理页面
2. 点击"新建角色"按钮
3. 填写角色编码、名称、描述、排序
4. 选择权限列表
5. 点击"确认"保存
6. 系统提示"创建成功"
7. 新角色出现在列表中

备选流程 A：编码重复
3a. 填写的编码已存在 → 系统提示"编码已存在" → 返回步骤3

备选流程 B：名称重复
3b. 填写的名称已存在 → 系统提示"名称已存在" → 返回步骤3

备选流程 C：未选择权限
4c. 未选择任何权限 → 警告确认 → 继续保存
```

### 5.2 场景测试用例

| 用例编号 | 场景 | 步骤描述 | 预期结果 |
|---------|------|---------|---------|
| TC-SC-01 | 基本流程 | 正确填写所有字段并保存 | 成功创建角色 |
| TC-SC-02 | 备选流程A | 输入重复编码后保存 | 提示编码重复 |
| TC-SC-03 | 备选流程B | 输入重复名称后保存 | 提示名称重复 |
| TC-SC-04 | 备选流程C | 不选权限直接保存 | 成功创建(无权限) |

## 6. 接口测试用例

### 6.1 创建角色接口

| 接口 | POST /user/role |
|------|-----------------|
| Content-Type | application/json |

**请求示例：**
```json
{
  "code": "EDITOR",
  "name": "编辑员",
  "description": "负责内容编辑",
  "sort": 10,
  "enable": true,
  "permissionIds": [1, 2, 3]
}
```

**测试用例：**

| 用例编号 | 测试项 | 请求数据 | 预期响应码 | 预期消息 |
|---------|-------|---------|-----------|---------|
| TC-API-01 | 正常创建 | 完整有效数据 | 200 | 创建成功 |
| TC-API-02 | 缺少编码 | code 为空 | 400 | 角色编码不能为空 |
| TC-API-03 | 缺少名称 | name 为空 | 400 | 角色名称不能为空 |
| TC-API-04 | 编码重复 | code="SUPER_ADMIN" | 500 | 编码已存在 |

### 6.2 更新角色接口

| 接口 | PATCH /user/role/{id} |
|------|----------------------|
| Content-Type | application/json |

**测试用例：**

| 用例编号 | 测试项 | 请求 | 预期响应码 | 预期消息 |
|---------|-------|------|-----------|---------|
| TC-API-05 | 正常更新 | PUT /user/role/3 | 200 | 更新成功 |
| TC-API-06 | 角色不存在 | PUT /user/role/999 | 404 | 资源不存在 |
| TC-API-07 | 禁用系统角色 | enable=false (id=1) | 500 | 系统角色不能禁用 |

### 6.3 删除角色接口

| 接口 | DELETE /user/role/{id} |
|------|------------------------|

**测试用例：**

| 用例编号 | 测试项 | 请求 | 预期响应码 | 预期消息 |
|---------|-------|------|-----------|---------|
| TC-API-08 | 正常删除 | DELETE /user/role/4 | 200 | 删除成功 |
| TC-API-09 | 删除系统角色 | DELETE /user/role/1 | 500 | 系统角色不能删除 |
| TC-API-10 | 删除有用户的角色 | DELETE /user/role/2 | 500 | 角色下有用户 |

### 6.4 查询角色接口

| 接口 | GET /user/role/page |
|------|---------------------|

**参数：**
- pageNo: 页码 (默认1)
- pageSize: 每页条数 (默认10)
- name: 角色名 (可选)
- enable: 启用状态 (可选)

**测试用例：**

| 用例编号 | 测试项 | 请求参数 | 预期结果 |
|---------|-------|---------|---------|
| TC-API-11 | 默认查询 | 无参数 | 返回第1页10条 |
| TC-API-12 | 按名称筛选 | name=管理 | 返回包含"管理"的角色 |
| TC-API-13 | 按状态筛选 | enable=true | 只返回启用的角色 |
| TC-API-14 | 分页查询 | pageNo=2&pageSize=5 | 返回第2页5条 |
